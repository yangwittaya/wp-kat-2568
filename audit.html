แนวการตรวจสอบ

โครงการจัดซื้อครุภัณฑ์ห้องปฏิบัติการ IoT Smart Farm
วิทยาลัยเกษตรและเทคโนโลยีศรีสะเกษ — สำนักตรวจเงินแผ่นดินจังหวัดศรีสะเกษ

แนวการตรวจสอบ
1. ข้อมูลโครงการ
หน่วยงานที่ตรวจสอบ สำนักตรวจเงินแผ่นดินจังหวัดศรีสะเกษ ปีงบประมาณ พ.ศ. 2568
หน่วยรับตรวจ วิทยาลัยเกษตรและเทคโนโลยีศรีสะเกษ วงเงินสัญญา 4,987,000 บาท
ชื่อโครงการ โครงการจัดซื้อครุภัณฑ์ห้องปฏิบัติการเทคโนโลยีอุตสาหกรรมการทำฟาร์มอัจฉริยะเกษตรกรรม IoT ด้วยระบบ APP จำนวน 1 ชุด
รายการทั้งหมด 19 รายการ รายการที่ตรวจเฉพาะ รายการที่ 12, 13, 16, 17 และ 19 (ความเสี่ยงสูง)
เลขที่สัญญา วันที่ทำสัญญา
2. วัตถุประสงค์การตรวจสอบ

การตรวจสอบครั้งนี้มีวัตถุประสงค์เพื่อ:

  1. ตรวจสอบการตรวจรับพัสดุตามสัญญาว่าตรงคุณลักษณะเฉพาะใน TOR ครบถ้วนหรือไม่
  2. ตรวจสอบกระบวนการตรวจรับว่าดำเนินการถูกต้องตาม พ.ร.บ. การจัดซื้อจัดจ้างและการบริหารพัสดุภาครัฐ พ.ศ. 2560
  3. ตรวจสอบคุณภาพพัสดุโดยเน้นรายการที่มีความซับซ้อนทางเทคนิคและมีความเสี่ยงสูง
3. ขอบเขตการตรวจสอบ

การตรวจสอบครั้งนี้ครอบคลุมการตรวจรับพัสดุโครงการจัดซื้อครุภัณฑ์ห้องปฏิบัติการ IoT Smart Farm วงเงิน 4,987,000 บาท จำนวน 19 รายการ โดยทำการตรวจสอบเฉพาะรายการที่มีความเสี่ยงสูง ได้แก่ รายการที่ 12, 13, 16, 17 และ 19 ซึ่งมีคุณลักษณะเฉพาะทางเทคนิคซับซ้อน ต้องการการทดสอบจริง และมีมูลค่ารวมสูง

รายการที่ไม่ได้ตรวจ (1–11, 14, 15, 18): เป็นอุปกรณ์มาตรฐานทั่วไปที่ตรวจรับโดยคณะกรรมการตรวจรับพัสดุของหน่วยรับตรวจแล้ว
4. รายการที่มุ่งตรวจสอบ (ความเสี่ยงสูง)
ลำดับรายการที่ชื่อรายการจำนวนระดับความเสี่ยงเหตุผล
112ชุดทดลองควบคุมการทำงานระบบสมาร์ทฟาร์ม3 ชุดสูงมากPLC/HMI/Gateway/Cloud ซับซ้อนสูง
213ชุดซอฟต์แวร์ Dashboard IoT1 ชุดสูงมากต้องมี ISO 9001 Scope เฉพาะ + ทดสอบ ≥29 ฟีเจอร์
316ระบบเลี้ยงไก่ AI Agriculture Chicken Tracking1 ชุดสูงมากโรงเรือน+AI Camera+เซ็นเซอร์+ระบบควบคุม มูลค่าสูง 1,795,000 บาท
417ระบบเลี้ยงสัตว์แบบแนวตั้ง RAS1 ชุดสูงมากระบบ RAS ซับซ้อน โรงเรือน+น้ำหมุนเวียน มูลค่า 1,200,000 บาท
519ระบบ E-Learning1 ระบบสูงมากต้องมี ISO 9001 + เครื่องหมายการค้า + LOA + เนื้อหา 4 ประเภท
5. ขั้นตอนการตรวจสอบ
5.1 รายการที่ 12 — ชุดทดลองควบคุมฯ สมาร์ทฟาร์ม
ข้อขั้นตอนวัตถุประสงค์หลักฐาน
12.1ตรวจนับจำนวน 3 ชุดยืนยันจำนวนครบถ้วนรายงานตรวจรับ, ทะเบียนพัสดุ
12.2ทดสอบ PLC ≥14 I/O + RS485/Modbusยืนยัน PLC ตามสเปคแค็ตตาล็อก, ทดสอบสด, ภาพถ่าย
12.3ทดสอบ HMI ≥7 หน้า แสดงผลครบยืนยัน HMI ทำงานครบแค็ตตาล็อก, บันทึกหน้าจอ
12.4ตรวจ Node Gateway สเปค (CPU/RAM/WiFi/BLE)ยืนยัน Node Gateway ตรงสเปคแค็ตตาล็อก, บันทึกการเชื่อมต่อ
12.5ตรวจ Local Cloud สเปค (CPU/Gigabit/BT5/HDMI 4K)ยืนยัน Local Cloud ตรงสเปคแค็ตตาล็อก, ภาพ System Info
12.6ทดสอบสื่อสาร RS485/Modbus ครบวงจรยืนยันระบบสื่อสารทำงานบันทึก Log, ภาพหน้าจอ Dashboard
12.7ตรวจ Relay/Hub-LAN/แหล่งจ่ายไฟยืนยันครบและทำงานได้รายงานตรวจรับ, ภาพถ่าย
5.2 รายการที่ 13 — ชุดซอฟต์แวร์ Dashboard IoT
ข้อขั้นตอนวัตถุประสงค์หลักฐาน
13.1ตรวจ ISO 9001:2015 Scope เฉพาะ + Weblinkยืนยัน ISO ตรง Scope และยังบังคับใช้ใบรับรอง ISO, บันทึกหน้าจอ Weblink
13.2ตรวจหนังสือแต่งตั้งตัวแทน (LOA)ยืนยันสิทธิ์ตัวแทนในไทยLetter of Authorization
13.3ตรวจแค็ตตาล็อก (ภาพจริง/จากโรงงาน)ยืนยันแค็ตตาล็อกตรงข้อกำหนดแค็ตตาล็อก Dashboard IoT
13.4ทดสอบฟังก์ชัน Dashboard ≥7 ข้อหลักยืนยันทำงานได้ตามฟีเจอร์บันทึกผล, ภาพหน้าจอทุกฟีเจอร์
13.5ทดสอบโปรแกรมพัฒนา Nodeยืนยันโปรแกรมพัฒนาใช้ได้จริงบันทึกผล, ภาพหน้าจอโปรแกรม
5.3 รายการที่ 16 — ระบบเลี้ยงไก่ AI Agriculture Chicken Tracking
ข้อขั้นตอนวัตถุประสงค์หลักฐาน
16.1วัดโรงเรือน ≥6×12×4 ม. + ตรวจวัสดุยืนยันขนาดและวัสดุตามสัญญาภาพถ่าย, บันทึกวัดขนาด
16.2วัด Buffer Room ≥2×6 ม.ยืนยัน Buffer Room ตรงขนาดภาพถ่าย, บันทึกวัดขนาด
16.3ตรวจพื้นคอนกรีต ≥8×14×0.08 ม.ยืนยันพื้นคอนกรีตตรงขนาดบันทึกวัดพื้นที่และความหนา
16.4ตรวจตู้ควบคุม/Gateway/เซ็นเซอร์ 5 ชนิดยืนยันครบและทำงานได้จริงแค็ตตาล็อก, ใบรับรอง ISO, บันทึกผลทดสอบ
16.5ทดสอบกล้อง AI ตรวจสุขภาพไก่ยืนยัน AI ทำงานได้จริงบันทึก VDO/ภาพ AI, ภาพผลลัพธ์
16.6ทดสอบระบบน้ำพ่นหมอก + อาหาร-น้ำ 2 แถวยืนยันระบบทำงานครบบันทึก VDO, รายงานตรวจรับ
16.7ตรวจตู้อนุบาล 2 ชุด + ตู้ฟักไข่ 1 ชุด + พัดลม + CCTVยืนยันครบและทำงานได้ภาพถ่าย, บันทึกผลทดสอบ
16.8ตรวจ ISO 9001 ของ Gateway และซอฟต์แวร์ยืนยันใบรับรองครบและถูกต้องใบรับรอง ISO ทุกรายการ, บันทึก Weblink
5.4 รายการที่ 17 — ระบบ RAS เลี้ยงสัตว์แบบแนวตั้ง
ข้อขั้นตอนวัตถุประสงค์หลักฐาน
17.1วัดโรงเรือน ≥4×12×4 ม. + ตรวจวัสดุยืนยันขนาดและวัสดุภาพถ่าย, บันทึกวัดขนาด
17.2ตรวจพื้นคอนกรีต ≥6×14×0.08 ม.ยืนยันพื้นคอนกรีตตรงขนาดบันทึกวัดพื้นที่และความหนา
17.3ตรวจนับกล่องเพาะ 4 ชุด 100 ใบยืนยันจำนวนครบถ้วนรายงานตรวจรับ, ภาพถ่าย
17.4ทดสอบระบบ RAS (กรองทราย/Skimmer/โอโซน/UV/Pump)ยืนยัน RAS ทำงานสมบูรณ์บันทึก VDO เดินระบบ
17.5ทดสอบเซ็นเซอร์คุณภาพน้ำ (PH/EC/DO/Temp)ยืนยันเซ็นเซอร์วัดค่าแม่นยำบันทึกค่าที่วัดได้/ค่าอ้างอิง
17.6ตรวจ ISO 9001 Gateway + Weblinkยืนยันใบรับรอง ISO ถูกต้องใบรับรอง ISO Gateway, บันทึก Weblink
17.7ตรวจแคลมป์มิเตอร์ดิจิตอล + ไฟ LED 12 ชุดยืนยันครบและทำงานได้ภาพถ่าย, บันทึกผลทดสอบ
5.5 รายการที่ 19 — ระบบ E-Learning
ข้อขั้นตอนวัตถุประสงค์หลักฐาน
19.1ตรวจ ISO 9001 Scope เฉพาะ + Weblinkยืนยัน ISO ตรง Scope และบังคับใช้ใบรับรอง ISO, บันทึก Weblink
19.2ตรวจเครื่องหมายการค้าจดทะเบียนยืนยันซอฟต์แวร์มีเครื่องหมายการค้าใบจดทะเบียนเครื่องหมายการค้า
19.3ตรวจหนังสือแต่งตั้งตัวแทน E-Learning (LOA)ยืนยันสิทธิ์ตัวแทนในไทยLetter of Authorization
19.4ทดสอบ LMS ครบ 7 ฟีเจอร์ยืนยัน LMS ทำงานครบฟีเจอร์บันทึกผล, ภาพหน้าจอทุกฟีเจอร์
19.5ตรวจเนื้อหาบทเรียน (Text/Graphic/Video/3D)ยืนยันเนื้อหาครบตามสัญญารายการเนื้อหา, บันทึกหน้าจอ
19.6ทดสอบระบบ Backup + Restoreยืนยัน Backup ทำงานได้บันทึกผลทดสอบ
19.7ทดสอบบน Computer/Smartphone/Tabletยืนยัน Responsive ทุก Platformบันทึกผลบนอุปกรณ์จริง
6. หลักฐานที่ต้องรวบรวม
ลำดับประเภทหลักฐานรายละเอียดหมายเหตุ
1สัญญาจัดซื้อสัญญา + TOR แนบท้ายรวบรวมก่อนออกตรวจ
2รายงานการตรวจรับพัสดุรายงาน + ลายเซ็นครบถ้วนขอจากหน่วยรับตรวจ
3ใบรับรอง ISO 9001:2015ของผู้ผลิตรายการ 13/16/17/19 + Weblinkตรวจสอบ Weblink ออนไลน์
4หนังสือแต่งตั้งตัวแทนจำหน่ายLetter of Authorization รายการ 13 และ 19ขอจากผู้ขาย/หน่วยรับตรวจ
5แค็ตตาล็อกจากโรงงานPLC/HMI/Node Gateway/Dashboard IoTตรวจสอบแหล่งที่มา
6บันทึกผลการทดสอบทุกรายการความเสี่ยงสูง + ภาพถ่าย/VDOจัดทำระหว่างออกตรวจ
7ใบส่งของ/ใบแจ้งหนี้เอกสารส่งมอบจากผู้ขายขอจากหน่วยรับตรวจ
8ทะเบียนพัสดุบันทึกรายการพัสดุที่ได้รับขอจากหน่วยรับตรวจ
9เครื่องหมายการค้าจดทะเบียนของซอฟต์แวร์ E-Learning รายการ 19ขอจากผู้ขาย/หน่วยรับตรวจ
